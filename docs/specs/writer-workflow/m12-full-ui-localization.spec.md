# M12.5 Complete Spanish UI Localization

Status: specified

## Purpose

Make the complete DreamJotter macOS interface usable by Spanish-speaking writers in Mexico and Latin America, with no workflow requiring English comprehension.

M12.5 completes the application-localization portion begun in M12.4. It covers every visible string, generated message, menu, dialog, validation result, accessibility label, export status, backup and restore workflow, library state, editor control, review finding, diagnostic presentation, and empty state in the macOS app.

Screenplay content is never translated. Application language and screenplay language remain independent.

## Supported Locales

Required shipping localizations:

- `en` — canonical source language;
- `es-MX` — primary reviewed Spanish translation;
- `es-419` — Latin American Spanish translation and fallback.

`es-MX` and `es-419` may share text where terminology is region-neutral. Differences must be explicit where Mexican usage materially improves clarity.

## Definition of Fully Localized

The Spanish UI is complete only when all of the following are true:

1. Every user-visible English literal has a localization key.
2. Every key used by the macOS target has both `es-MX` and `es-419` translations.
3. No localization key, raw enum value, implementation identifier, or unintended English fallback is visible in normal operation.
4. Dynamic messages use localized format strings, plural rules, and ordered substitutions rather than concatenated English fragments.
5. Errors and diagnostics display localized messages generated from stable codes.
6. Menus, dialogs, panels, tooltips, accessibility labels, and VoiceOver hints are localized.
7. The interface remains readable without clipping at the minimum supported window size.
8. Search, keyboard shortcuts, file formats, and screenplay semantics behave identically in English and Spanish.
9. The writer can create, edit, save, reopen, review, export, back up, restore, and close a project without encountering required English text.

## Localization Architecture

- Use a single String Catalog for the macOS target unless a framework-specific catalog is required.
- Prefer stable semantic keys such as `workspace.characters.title` rather than using full English sentences as permanent identifiers.
- Existing literal-based keys may remain temporarily, but M12.5 acceptance requires migration or an explicit compatibility mapping.
- Use `LocalizedStringResource` or `LocalizedStringKey` for SwiftUI presentation.
- Use `String(localized:)` for generated model or controller messages.
- Never localize persisted enum raw values, package keys, diagnostic codes, filenames, file extensions, or interchange syntax.
- Never persist a localized user-facing sentence as the only representation of an error or finding.
- Locale changes must update visible UI without mutating project data.

## Required Source Audit

The implementation must inspect every Swift file under `Apps/DreamJotterMac` and classify all user-visible strings from:

- `Text`;
- `Label`;
- `Button`;
- `TextField` and `SecureField` placeholders;
- `Picker` labels and options;
- alerts and confirmation dialogs;
- sheet titles;
- `navigationTitle`;
- menu and command labels;
- `.help` tooltips;
- accessibility labels, values, and hints;
- panel titles and suggested filenames;
- empty states;
- status and error messages generated by view models;
- formatted counts and summaries;
- export, backup, restore, and review feedback.

The audit must also inspect user-facing messages originating from `DreamJotterCore` that are displayed by the macOS app.

## Grammar and Pluralization

String Catalog variations must cover at least these pluralized concepts:

- 0, 1, and many scenes;
- characters;
- locations;
- notes;
- TODOs;
- findings;
- search matches;
- affected screenplay elements;
- selected notes;
- orphaned links.

Examples:

- `0 escenas`, `1 escena`, `%lld escenas`;
- `0 coincidencias`, `1 coincidencia`, `%lld coincidencias`;
- `1 nota seleccionada`, `%lld notas seleccionadas`.

Do not construct these forms with manual singular/plural suffixes.

## Dynamic Message Rules

- Use format arguments with positional meaning documented in the catalog.
- Never concatenate a translated noun with an untranslated count or status.
- Dates, times, lists, and numbers use locale-aware formatters.
- File paths, filenames, identifiers, and format names may remain verbatim inside localized surrounding text.
- Error technical details may remain English only when hidden behind a developer-oriented disclosure; the primary user message must be localized.

## Accessibility Localization

Every interactive control requires localized accessibility text when its visible label is insufficient.

Required coverage includes:

- icon-only previous/next search buttons;
- scene move buttons;
- result-selection controls;
- editor navigation;
- severity indicators;
- export-state icons;
- profile detection actions;
- note selection checkboxes;
- settings controls.

VoiceOver must not read raw symbol names, enum raw values, or localization keys.

## Layout Requirements

- Spanish text must not be truncated at the minimum supported window size.
- Buttons may grow horizontally or wrap where appropriate.
- Dialogs must support longer Spanish sentences.
- Fixed widths must be reviewed and widened or made adaptive.
- Toolbar labels and segmented controls must remain readable.
- Dynamic Type and accessibility text sizes must preserve access to primary actions.

## Automated Enforcement

Add a localization audit script that fails when:

- a user-visible SwiftUI literal is absent from the catalog;
- an English key lacks `es-MX` or `es-419` translation;
- a translation is marked `new`, `needs_review`, or empty;
- a normal Spanish UI smoke test renders a key-like identifier;
- a required accessibility label is missing;
- a prohibited raw enum value is shown directly.

Recommended command:

```text
python3 scripts/localization-check --target Apps/DreamJotterMac --locales es-MX,es-419
```

The script must support a documented allowlist for product names, file formats, keyboard symbols, and genuinely developer-only text.

## Executable and UI Test Coverage

Required automated tests:

1. Every catalog key has English, `es-MX`, and `es-419` values.
2. Application-language override persists and updates the locale environment.
3. Every workspace can be opened under `es-MX` without missing keys.
4. Every alert and confirmation path has a Spanish string.
5. Singular and plural count strings render correctly.
6. Error codes resolve to Spanish user messages.
7. Export and restore feedback renders in Spanish.
8. Search, selection, and navigation behavior is unchanged by locale.
9. Accessibility labels exist for icon-only controls.
10. Minimum-width snapshot or UI tests detect clipped primary controls.
11. No unintended English sentence appears in the Spanish smoke-test allowlist report.
12. Locale switching never mutates project or screenplay content.

## Manual Acceptance Journey

Complete this full workflow using only Spanish UI:

1. Launch DreamJotter and select Spanish.
2. Create a project.
3. Enter title, premise, and synopsis.
4. Write and edit a Spanish screenplay.
5. Search and navigate matches.
6. Create and edit character and location profiles.
7. Convert and ignore detected entities.
8. Edit scene cards and planning order.
9. Add, edit, resolve, reopen, and delete notes.
10. Navigate from a note or review finding to the script.
11. Review health findings.
12. Export Fountain and PDF.
13. Create a JSON backup.
14. Save and close the project.
15. Reopen it from Recent Projects.
16. Restore the backup.
17. Close with unsaved changes and exercise Save, Discard, and Cancel.

No required step may display English-only instructions or controls.

## Acceptance Exit Criteria

M12.5 is accepted only when:

- the localization audit reports zero missing Spanish translations;
- all automated tests pass;
- the full manual journey succeeds in `es-MX` and `es-419`;
- no blocking truncation or inaccessible control is found;
- a native Spanish speaker approves terminology and tone;
- the README and release notes accurately describe full Spanish UI support.

## Out of Scope

- machine translation of screenplay content;
- translation of user-entered names, dialogue, notes, locations, or metadata;
- languages other than English and Spanish;
- dubbing, subtitle translation, or screenplay adaptation;
- localization of technical logs intended only for developers.
